# 📖 The Vibe Lexicon

Technology is full of confusing, threatening-sounding words. When you are directing an AI, it helps to understand what the AI is actually talking about when it throws these words back at you.

Here is how we translate angry computer words into human concepts.

---

| Tech Jargon | What it Actually Means | Church/Community Analogy |
| :--- | :--- | :--- |
| **Frontend** | The part of the app the user clicks and sees (buttons, colors, text, layouts). | **The Lobby/Sanctuary:** The chairs, the lighting, the signs. It's the environment people interact with. |
| **Backend** | The servers and logic happening out of sight. | **The Operations Room/Admin Office:** Working hard behind closed doors to make sure the service happens smoothly. |
| **Database** | A system designed specifically for storing information permanently. | **The Filing Cabinet/Registry:** Where we keep the important documents safe from fire. |
| **State** | The current status of the app at this exact millisecond (e.g., is the menu open or closed? Is the user logged in?). | **The Light Switch:** A switch only has two states: On or Off. The app has to remember if the switch is currently flipped or not. |
| **Component** | A reusable block of code (like a button, a nav bar, or a card). React is built entirely out of these blocks. | **Lego Bricks/Modular Staging:** You build one standard stage box, and then you use 10 of them to build the exact stage size you need. |
| **API (Application Programming Interface)** | A messenger that takes requests, tells a system what you want to do, and returns the response back to you. | **The Usher:** You ask the usher where the bathroom is; the usher holds that information and hands it to you. You don't need a map of the whole building yourself. |
| **Bug** | An error in logic or syntax that causes the app to crash or behave weirdly. | **A Misprinted Bulletin:** The bulletin says Service starts at 9am, but it’s actually 10am. People are confused. We need to print a new one. |
| **Deploy** | Putting your app on a public server so anyone can access it via a web link. | **Opening the Doors:** The building is finished, the chairs are set, it's time to unlock the front doors on Sunday morning. |
| **Auth (Authentication)** | Verifying the identity of a person before letting them in. | **The Bouncer / The Check-In Desk:** "Are you on the list? Please show me your ID." |
| **Repo (Repository)** | A storage space (usually GitHub) where your code lives and is tracked over time. | **The Master Binder/Archive:** The historical record of every decision and blueprint used to build the church. |
| **Vite / React / Tailwind** | The specific tools we use to build the Frontend. | **The Hammer, The Wood, and The Paint:** Our chosen instruments for construction. |

When the AI gives you an error saying "The Component failed to render state properly," you now know it means: *"The Lego brick forgot if its light switch was turned on or off."* 

Much less scary!
