# 🤝 Collaboration Guide (Playing in the Sandbox)

Eventually, you will want to build an application with another person from your church group. 

If you both edit the same file on `main` at the same time and hit push, you will run into terrifying red errors called "Merge Conflicts." 

To avoid this, we use **Branches**.

---

## 🌿 The Concept of Branches (Parallel Universes)

The `main` branch is your sacred, working, production-ready code. You should ideally never experiment directly on `main`.

Instead, you create a **Branch**. A branch is an exact copy of `main`, but in a parallel universe. You can break things in your branch, and the `main` branch is entirely unaffected.

### Step-by-Step Branching (The Agentic Way):

**1. Let Antigravity Create the Branch:**
You do not need to memorize terminal commands for this. Simply tell Antigravity:
> "Please create a new git branch called `featured-announcements` and switch to it."

**2. Do the Work:**
Let Antigravity build the feature in this new universe. Review the code, and test it in your browser.

**3. Let Antigravity Push the Branch:**
Once you are happy, tell Antigravity:
> "Please commit these changes with the message 'Added announcements' and push this branch to GitHub."

---

## 📥 The Pull Request (The Proposal)

Your new feature works beautifully in its parallel universe, but how do we get it into the safe `main` timeline? We don't force it. We politely request it.

1. Go to your repository on **GitHub.com**.
2. You will see a big green button that says **"Compare & pull request"** next to your recently pushed branch.
3. Click it.
4. Give your Request a title: *"Implemented the new Featured Announcements widget."*
5. Click **"Create pull request"**.

**This is where Collaboration happens.** 
Your teammate can now look at your Pull Request on GitHub, review the exact lines of code you added, leave comments, and finally click **Merge** to bring your feature into the `main` project.

By using Branches and Pull Requests, teams of 2 or 2,000 can work on the same codebase simultaneously without stepping on each other's toes!
